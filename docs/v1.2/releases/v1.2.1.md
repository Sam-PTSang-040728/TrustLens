## Implemented In This Patch

- Public registration policy is configurable with `PUBLIC_REGISTRATION_ENABLED`.
- Public registration defaults to enabled only in development.
- Closed registration returns stable error code `AUTH_PUBLIC_REGISTRATION_DISABLED`.
- Frontend registration uses backend-aligned password minimum of 10 characters.
- Frontend login/register display structured API error messages and correlation IDs.
- Frontend version is bumped to `1.2.1`.
- Production mock-mode guard remains enforced by `vite.config.js`.

## Unchanged Compatibility Surface

- API prefix remains `/api/v1`.
- Trust Score remains `trust-score-v1.2`.
- C1-C7 criteria, scoring weights, thresholds, and calibration logic are unchanged.
- No OCR, batch processing, LMS integration, WebSocket/SSE, distributed broker, or multi-tenancy is introduced.
