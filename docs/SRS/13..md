## 13.3. Quy trình vận hành job

34. API nhận upload và tạo submission.

35. API tạo processing_job với status QUEUED.

36. Worker nhận job và cập nhật progress theo từng bước.

37. Nếu lỗi tạm thời, worker retry theo cấu hình.

38. Nếu lỗi vĩnh viễn, worker ghi error_code và chuyển FAILED.

39. Frontend polling /api/jobs/{id} để cập nhật UI.

40. Khi COMPLETED, frontend gọi report endpoint để hiển thị kết quả.
